# On-Chain Monitoring Guide: Crypto Wallet Honeypot

This document explains how to set up real-time monitoring for the blockchain addresses generated by your honeypots. By watching these addresses on-chain, you can detect when an attacker has successfully exfiltrated a private key and is attempting to use it.

## Overview

The Crypto Wallet Honeypot system provides two layers of detection for on-chain activity:
1. **Passive Watchlists:** Using free services (like Etherscan or Solscan) to receive email or webhook notifications.
2. **Active Monitoring:** Using the `chain-monitor` service (if deployed) to ingest on-chain events directly into Wazuh.

---

## 1. Exporting Honeypot Addresses

Before setting up monitoring, you need to extract the public addresses from your encrypted manifest.

```bash
# Export to a JSON file
honeypot-deployer export-addresses \
  --manifest ./honeypot-artifacts/manifest.json \
  --output ./my-addresses.json
```

The resulting `my-addresses.json` will look like this:
```json
{
  "btc": ["1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa"],
  "eth": ["0x742d35Cc6634C0532925a3b844Bc454e4438f44e"],
  "sol": ["v6S3Z9X5u6T6zX5u6T6zX5u6T6zX5u6T6zX5u6T6z"]
}
```

## 2. Setting Up Block Explorer Watchlists

This is the easiest way to get notified of any activity. Most major block explorers offer a "Watchlist" or "Address Alert" service.

### Ethereum & EVM Chains (Etherscan)
1. Create a free account on [Etherscan.io](https://etherscan.io/).
2. Navigate to **Account** -> **Watch List**.
3. Click **Add** and paste your exported ETH address.
4. Set **Notification Method** to "Email" or "Webhook".
5. Repeat for other EVM chains (BSCSan, PolygonScan, etc.) if you deployed those.

### Bitcoin (Blockchain.com / BlockCypher)
1. Use a service like [BlockCypher](https://www.blockcypher.com/dev/bitcoin/) or [Blockchain.com's API](https://www.blockchain.com/explorer/api/blockchain_api) to set up webhooks for your BTC addresses.

### Solana (Solscan)
1. Create an account on [Solscan.io](https://solscan.io/).
2. Use the **Solana Account Tracking** feature to add your honeypot addresses.

## 3. Integrating with Wazuh

If you are using the optional `chain-monitor` service, you can feed these alerts directly into your Wazuh dashboard.

### 1. Configure the Chain Monitor
Point your chain monitor daemon to the `my-addresses.json` file generated in Step 1.

### 2. Enable Log Ingestion
Ensure your Wazuh Manager is configured to read the chain monitor's event log. Add this to your manager's `ossec.conf`:

```xml
<localfile>
  <log_format>json</log_format>
  <location>/var/log/chain-monitor/events.json</location>
</localfile>
```

### 3. High-Severity Rules
The default honeypot rules (100530 and 100532) are designed to trigger on these on-chain events:
- **Rule 100530 (Level 15):** On-chain activity detected on a honeypot address.
- **Rule 100532 (Level 15):** Outbound transfer from a honeypot address.

---

## Security Best Practices

- **NEVER** deposit real funds into these addresses. They are meant to be empty.
- **NEVER** use these addresses for any legitimate purpose.
- If you receive an on-chain alert, **assume the endpoint where that honeypot was deployed is fully compromised.**
