# On-Chain Monitoring Guide: Crypto Wallet Honeypot

On-chain monitoring is Layer 4 of the Crypto Wallet Honeypot's detection strategy. It allows you to detect if an attacker successfully imports a stolen key and attempts to use it on the blockchain.

## Overview

The honeypot keys generated by the `honeypot-deployer` are real, valid keys, but they do not contain any funds. When an attacker steals these keys, their first action is often to import them into a wallet or use a script to check for balances. By placing the corresponding public addresses on a blockchain "watchlist," you can receive alerts the moment any activity occurs on those addresses.

## Step 1: Export Honeypot Addresses

Use the CLI to export all public addresses generated for your deployment into a JSON file:

```bash
honeypot-deployer export-addresses \
  --manifest ./honeypot-artifacts/manifest.json \
  --output ./chain-monitor-addresses.json
```

## Step 2: Set Up Block Explorer Watchlists

Most major block explorers provide a "watchlist" or "address alert" service for free (with registration).

### Recommended Explorers by Chain:

| Chain | Recommended Explorer | Features |
|-------|----------------------|----------|
| **Bitcoin** | [Blockchain.com](https://www.blockchain.com/explorer) or [Mempool.space](https://mempool.space/) | Email alerts on tx |
| **Ethereum / EVM** | [Etherscan](https://etherscan.io/) | Detailed email/webhook alerts |
| **Solana** | [Solscan](https://solscan.io/) | Address tracking |
| **XRP** | [XRP Scan](https://xrpscan.com/) | Account monitoring |
| **Cardano** | [Cardanoscan](https://cardanoscan.io/) | Address alerts |

### Setting Up a Watchlist (e.g., Etherscan):
1. Create a free account on [Etherscan.io](https://etherscan.io/).
2. Navigate to **My Account** -> **Watchlist**.
3. Click **Add** and paste an address from your `chain-monitor-addresses.json`.
4. Select **Notify on Incoming & Outgoing Tx**.
5. Repeat for all Ethereum addresses in your export.

## Step 3: Integrating with Wazuh (Advanced)

For professional SOC environments, you can integrate these block explorer alerts back into Wazuh:

1. **Webhooks:** Use explorers that support webhooks (like Etherscan's developer API or specialized services like Alchemy/QuickNode).
2. **Log Collector:** Set up a small script (e.g., in Python) that polls the block explorer APIs for your addresses and writes any new transactions to a log file.
3. **Wazuh Monitoring:** Configure the Wazuh agent to monitor that log file.

### Example Integration Script Logic:
```python
# Pseudo-code for monitoring addresses
import requests
import json

addresses = json.load(open("chain-monitor-addresses.json"))

for chain, addrs in addresses.items():
    for addr in addrs:
        # Call block explorer API (e.g., Etherscan)
        response = requests.get(f"https://api.etherscan.io/api?module=account&action=txlist&address={addr}&apikey=YOUR_KEY")
        # If new transaction found, write to log
        if response.json()['result']:
            print(f"ALERT: On-chain activity detected on {chain} address {addr}!")
```

## Security Best Practices

- **Never Deposit Funds:** The honeypot addresses should never contain real cryptocurrency. Any activity on these addresses is a definitive sign of compromise.
- **Privacy:** Be aware that adding addresses to a public block explorer's watchlist associates them with your account on that explorer. Use a dedicated, non-identifiable account for maximum privacy.
