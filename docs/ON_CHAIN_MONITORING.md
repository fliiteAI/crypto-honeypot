# On-Chain Monitoring Guide: Tracking Honeypot Addresses

This guide explains how to track the generated honeypot addresses on the blockchain to detect when an attacker has imported your stolen keys and is attempting to use them.

## Exporting Honeypot Addresses

The first step is to export the public addresses generated by the honeypot deployer.

### Use the CLI to Export
Run the following command to generate a JSON file containing all monitored addresses from your manifest:

```bash
honeypot-deployer export-addresses \
  --manifest ./path/to/manifest.json \
  --output ./honeypot-watch-list.json
```

This JSON file lists the addresses for all supported chains (BTC, ETH, SOL, XRP, ADA).

## Setting Up Watchlists

Once you have your addresses, you can add them to popular block explorer watchlists to receive automated alerts.

### 1. Ethereum & EVM (Etherscan, Polyscan, etc.)
1. Create a free account on [Etherscan](https://etherscan.io/).
2. Navigate to **My Profile > Watch List**.
3. Add your honeypot ETH/ERC-20 addresses.
4. Select **Notify on Incoming & Outgoing Txns**.
5. Enable **Email Notifications**.

### 2. Solana (Solscan, Solana FM)
1. Use [Solscan](https://solscan.io/) or [Solana FM](https://solana.fm/).
2. Add your honeypot SOL addresses to your watchlist.
3. Configure alerts for any balance changes.

### 3. Bitcoin (Blockchain.com, Blockcypher)
1. Use [Blockchain.com's explorer](https://www.blockchain.com/explorer) or [Blockcypher](https://www.blockcypher.com/).
2. Set up address-level notifications for your BTC addresses.

## Automated Chain Monitoring Service

For larger deployments, we recommend using a dedicated chain monitoring service that integrates directly with Wazuh.

### Integration with Wazuh
A custom chain monitoring service can be configured to send JSON events to the Wazuh Manager's `log-collector`.

#### Example JSON Alert:
```json
{
  "source": "chain-monitor",
  "event_type": "honeypot_chain_activity",
  "chain": "ethereum",
  "address": "0x123...",
  "activity_type": "outbound_transfer",
  "details": "Transfer of 0 ETH to 0xAbc...",
  "timestamp": "2023-10-27T10:00:00Z"
}
```

This will trigger **Wazuh Rule 100530** (Level 15 Critical Alert).

### Critical Rules
- **100530:** General on-chain activity detected.
- **100531:** Balance query on honeypot address (attacker evaluation).
- **100532:** Outbound transfer attempt (active theft).
- **100533:** Token approval (DeFi drainer activity).

## Security Best Practices
- **Do NOT deposit real funds** to honeypot addresses. They are intended to be "empty" to maximize their effectiveness as bait without financial risk.
- **Monitor for Small "Gas" Injections:** Attackers may send a small amount of native tokens (e.g., 0.01 ETH) to a stolen address to cover gas fees for a subsequent transfer of other assets. This is a high-fidelity indicator of an active drain.
