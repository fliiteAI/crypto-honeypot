# On-Chain Monitoring: Tracking Honeypot Activity

On-chain monitoring is the final detection layer. By tracking the public addresses of your honeypots, you can confirm if an attacker has successfully stolen and imported your honeypot keys into their own wallet software.

## Exporting Honeypot Addresses

The first step is to export the public addresses generated by the `honeypot-deployer`. These addresses are safe to share and import into block explorer watchlists.

```bash
# Export all generated addresses from a manifest
honeypot-deployer export-addresses \
  --manifest ./honeypot-artifacts/manifest.json \
  --output ./chain-monitor-addresses.json
```

This will generate a JSON file containing the addresses for BTC, ETH, SOL, XRP, and ADA.

## Setting Up Watchlists

You can use popular block explorers to set up real-time email or webhook alerts when activity occurs on your honeypot addresses.

### 1. Ethereum & EVM (Etherscan)
- Log in to [Etherscan](https://etherscan.io).
- Navigate to **Account** > **Watch List**.
- Add each ETH address.
- Set **Notify on Incoming & Outgoing Txns**.
- (Optional) Apply to other EVM explorers like BscScan, Polygonscan, etc.

### 2. Bitcoin (Mempool.space)
- Use [mempool.space](https://mempool.space) to manually monitor or use an API-based service to watch for honeypot address activity.
- Look for **Input** activity from your honeypot addresses.

### 3. Solana (Solscan)
- Log in to [Solscan](https://solscan.io).
- Use the **Account Monitor** feature.
- Add your honeypot SOL addresses to receive alerts on any transaction activity.

## Automated Monitoring (Recommended)

For large deployments, we recommend using a dedicated on-chain monitoring service (e.g., Alchemy, QuickNode, or custom scripts) that can send webhooks directly to your Wazuh Manager or SOC.

### Detection Rule (Wazuh)
The custom Wazuh ruleset includes **Rule 100530** (On-chain activity on honeypot address) and **Rule 100532** (Outbound transfer from honeypot address). These can be triggered via logs from your on-chain monitoring service.

---

## What to Do on an On-Chain Alert?

1. **Immediate Endpoint Isolation:** If an on-chain alert occurs without a corresponding endpoint FIM alert, it may indicate a previously undetected compromise or an attacker using stolen credentials from a different environment.
2. **Review Forensic Snapshots:** Check if any forensic snapshots were triggered on your agents in the timeframe leading up to the on-chain activity.
3. **Monitor Attacker Wallet:** The address where the honeypot funds *would* have gone (if there were any) often belongs to the attacker. Use this address for further threat intelligence gathering.
